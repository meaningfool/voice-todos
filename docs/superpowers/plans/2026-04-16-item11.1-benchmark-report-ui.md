# Item 11.1 Benchmark Report UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** `docs/superpowers/specs/2026-04-16-item11.1-benchmark-report-ui-design.md`

**Goal:** Add a benchmark-owned, self-contained HTML report UI under `evals/reports/` and expose it through `benchmark report --html` and `benchmark report --open`.

**Architecture:** Keep the persisted JSON report as the canonical machine-readable artifact. Add a Python-side HTML orchestration layer under `evals/` that ensures the JSON report exists, builds or reuses a self-contained viewer shell, injects the current report JSON into that shell, and writes `evals/reports/<benchmark_id>.html`. Implement the viewer with the existing React/Vite/shadcn setup, but keep it out of the main app runtime and open it from the benchmark CLI only when requested.

**Tech Stack:** Python CLI/report orchestration, React 19, Vite, shadcn/ui, Tailwind CSS 4, pytest, browser validation via `agent-browser`
