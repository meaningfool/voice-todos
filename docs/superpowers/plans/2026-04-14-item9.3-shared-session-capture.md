# Item 9.3 Shared Session Capture For STT Curation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** `docs/superpowers/specs/2026-04-14-item9.3-shared-session-capture-design.md`

**Goal:** Make recorded STT sessions usable as a curation inbox for future STT eval cases by capturing provider-specific raw traces, sharing `sessions/recent/` across Conductor worktrees, and reserving `sessions/golden/` as tracked repo storage for promoted recordings.

**Architecture:** Keep session recording in `backend/app/`, keep Conductor worktree sharing in `scripts/conductor-setup.sh`, and keep the git storage contract explicit in `.gitignore`. This item is intentionally narrower than Item 10: it prepares the capture and promotion substrate, not the STT benchmark itself.

**Tech Stack:** Python 3.11+, FastAPI websocket session recording path, provider-specific STT adapters, pytest, shell-level setup and git-ignore verification

- [x] **Step 1: Record provider-specific trace filenames in the recorder**

Mistral connectors, and `MistralSession` exposes `raw_event_callback` so the
adapter can emit raw provider events into the recorder.

- [x] **Step 3: Start recordings with the active provider name**

The websocket flow now starts the recorder with the configured provider name so
the on-disk session layout matches the provider that actually ran.

## Phase 2: Share Recent Inbox And Track Golden Storage

**Behavioral delta:** Conductor worktrees can see the same `sessions/recent/`
inbox, while promoted recordings have a separate tracked home under
`sessions/golden/`.

### Task 2: Link the shared recent-session inbox and formalize the git storage split

**Files:**
- Modify: `scripts/conductor-setup.sh`
- Modify: `.gitignore`
- Create: `sessions/golden/.gitkeep`

- [x] **Step 1: Link `sessions/recent/` from the root checkout into each worktree**

`scripts/conductor-setup.sh` now creates the shared root-checkout
`sessions/recent/` directory when `CONDUCTOR_ROOT_PATH` is set and symlinks it
into the current worktree as `sessions/recent`.

- [x] **Step 2: Reserve tracked storage for promoted recordings**

`.gitignore` now keeps `sessions/recent/` ignored while explicitly allowing
`sessions/golden/` to be tracked, and `sessions/golden/.gitkeep` documents that
durable storage contract in the repo.

- [x] **Step 3: Define the acceptance procedures for the storage contract**

Acceptance procedure 1: shared recent-session inbox

```bash
tmp_root="$(mktemp -d)"
tmp_worktree="$(mktemp -d)"
mkdir -p "$tmp_root/backend" "$tmp_root/.codex/superpowers" "$tmp_root/sessions/recent" \
  "$tmp_worktree/backend" "$tmp_worktree/frontend" "$tmp_worktree/scripts"
cp scripts/conductor-setup.sh "$tmp_worktree/scripts/conductor-setup.sh"
(
  cd "$tmp_worktree"
  PATH="/usr/bin:/bin" CONDUCTOR_ROOT_PATH="$tmp_root" bash -c '
    uv() { :; }
    pnpm() { :; }
    export -f uv pnpm
    ./scripts/conductor-setup.sh
  '
)
test -L "$tmp_worktree/sessions/recent"
test "$(readlink "$tmp_worktree/sessions/recent")" = "$tmp_root/sessions/recent"
```

Expected:

- `sessions/recent` exists as a symlink in the worktree
- the symlink target is the shared root-checkout inbox

Acceptance procedure 2: git storage split

```bash
git check-ignore -v sessions/recent/probe.txt
if git check-ignore sessions/golden/.gitkeep >/dev/null; then
  echo "sessions/golden/.gitkeep should be tracked" >&2
  exit 1
fi
```

Expected:

- `sessions/recent/probe.txt` is ignored because of the `sessions/recent/` rule
- `sessions/golden/.gitkeep` is not ignored

- [x] **Step 4: Run supporting verification**

Run:

```bash
bash -n scripts/conductor-setup.sh
git diff --check -- .gitignore scripts/conductor-setup.sh
```

Expected: PASS

## Completion Notes

Item 9.3 is complete when:

- recent recorded sessions capture provider-specific raw traces through the
  shared recorder seam
- Soniox and Mistral both route raw events into that seam
- Conductor setup can expose one shared `sessions/recent/` inbox across
  worktrees
- `sessions/recent/` stays ignored while `sessions/golden/` is available for
  tracked promoted recordings

This item intentionally stops before:

- automatic promotion from `sessions/recent/` to `sessions/golden/`
- STT dataset rows
- benchmark locks
- STT benchmark execution
