# Item 9.1 Single-Stream Voxtral Adapter Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** `docs/superpowers/specs/2026-04-13-item9.1-voxtral-adapter-design.md`

**Goal:** Add a single-stream Voxtral-backed `SttSession` implementation that fits behind the Item 8 seam and makes the stop path rely on `transcription.done` rather than Soniox token-finalization semantics.

**Architecture:** Extend the shared STT contract just enough to expose an authoritative final transcript, add a dedicated `stt_mistral.py` adapter that maps additive Voxtral deltas into the app contract without faking Soniox semantics, and make `ws.py` consume the adapter-neutral final transcript on stop. Keep Soniox unchanged as the default path.

**Tech Stack:** Python 3.11+, `mistralai` realtime client, FastAPI websocket flow, pytest, existing STT abstraction in `backend/app/`

---

## Scope

This plan covers exactly three deliverables:

1. Extend the shared STT contract and provider-selection seam for a Voxtral session.
2. Implement the Voxtral realtime adapter and deterministic adapter tests.
3. Wire the adapter into `/ws` stop handling using an adapter-neutral final transcript accessor.

Out of scope for this plan:

- dual-delay
- delay tuning or benchmark work
- redesigning transcript rendering
- a new incremental extraction policy
- changing the default provider from Soniox
- adding any frontend or per-session STT provider selection control

---

## File Map

| File | Responsibility |
|------|----------------|
| `backend/app/stt.py` | Shared contract updates, including the final transcript accessor |
| `backend/app/stt_factory.py` | Provider selection for Soniox vs Mistral |
| `backend/app/stt_soniox.py` | Keep the default provider compliant with the updated shared contract |
| `backend/app/stt_mistral.py` | Voxtral adapter: session lifecycle, event translation, and stop semantics |
| `backend/app/ws.py` | Adapter-neutral stop path that uses the final transcript accessor if present |
| `backend/tests/test_stt_factory.py` | Focused provider-routing coverage for Soniox vs Mistral |
| `backend/tests/test_stt_soniox.py` | Regression coverage for the unchanged default provider contract |
| `backend/tests/test_stt_mistral.py` | Focused deterministic tests for Voxtral event translation and session behavior |
| `backend/tests/test_ws.py` | Stop-path and factory integration coverage with fake sessions |

Notes:

- Do not generalize session recording in this item. The current recorder is Soniox-shaped and can remain unchanged.
- The adapter should report missing Soniox semantics honestly rather than synthesizing them.
- The plan intentionally keeps Mistral transcript handling additive and simple.
- Provider choice remains config-driven in this item via `settings.stt_provider`; do not add query-string or `start`-message overrides.

This task adds the real adapter behavior:

- connect to Voxtral realtime
- translate additive deltas into `SttEvent`
- mark endpoint semantics unsupported
- resolve `wait_for_final_transcript()` on `transcription.done`
- expose `done.text` through `final_transcript_text`

- [ ] **Step 3: Implement `backend/app/stt_mistral.py`**

Add:

- the `final_transcript_text` accessor to `SttSession` in `backend/app/stt.py`
- a `final_transcript_text` property on `SonioxSession` in
  `backend/app/stt_soniox.py` that returns `None`
- `MISTRAL_CAPABILITIES = SttCapabilities(False, False)`
- a translation function for provider-native events
- `MistralSession` implementing `SttSession`
- `connect_mistral(...)`

Implementation guidance:

- `transcription.text.delta` -> `SttEvent(tokens=[SttToken(text=..., is_final=True)], finalization_state=NOT_OBSERVED, endpoint_state=UNSUPPORTED)`
- `transcription.done` should:
  - store `done.text` as the final transcript
  - set the final-transcript event
  - surface stream completion to the iterator
- `request_final_transcript()` should flush audio only
- `end_stream()` should end audio only
- do not invent endpoint markers or non-final tokens
- if `session.created` or `session.updated` appear, translate them into empty `SttEvent`s rather than transcript text
- make the realtime connection seam injectable enough for deterministic tests
  (for example via an injected connect/client factory)
- ignore `recorder`-style raw-message hooks on the Mistral path in this item;
  do not try to generalize session recording here
