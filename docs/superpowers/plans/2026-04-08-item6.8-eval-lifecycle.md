# Item 6.8 Eval Lifecycle Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Retrofit both eval runners so Pydantic `Experiment` metadata and Logfire become the canonical tracked-eval lifecycle, while the old local artifact plus enrichment flow is removed from the normal path.

**Architecture:** Add one shared experiment-identity helper that derives `experiment_name`, hybrid `experiment_id`, `execution_fingerprint`, and `batch_id` from a single config source of truth. Then migrate the transcript-only and incremental replay runners so Logfire uses dataset-first grouping plus human-readable experiment names, while the metadata payload carries the machine identity and exact execution fingerprint. Finish by deleting the legacy artifact path, rewriting the READMEs around dataset grouping and `batch_id`, and validating the new lifecycle with both automated tests and live smoke commands.

**Tech Stack:** Python 3.13, Pydantic Evals, Logfire, pytest, existing eval runners and dataset loaders

**Spec:** `docs/superpowers/specs/2026-04-07-item6.8-eval-result-structure-and-process-design.md`

---

## Prerequisites

- Read the 6.8 spec before changing code. The metadata contract and validation plan are the source of truth for scope.
- Keep Item 6.9 out of this implementation. Do not add benchmark metadata beyond the reserved optional fields.
- Do not build a new export format in this plan. The point of 6.8 is to remove the local artifact plus enrichment path from the normal tracked workflow, not to replace it with a different local canonical store.
- Use a dedicated Logfire dev project for the live smoke checks in Task 5.

Create `backend/evals/common/experiment_metadata.py` with a small surface:

```python
from __future__ import annotations

import hashlib
import json
import secrets
from datetime import UTC, datetime
from pathlib import Path
from typing import Any, Iterable


def build_batch_id(now: datetime | None = None) -> str:
    timestamp = (now or datetime.now(UTC)).strftime("%Y-%m-%dT%H-%M-%SZ")
    return f"{timestamp}-{secrets.token_hex(4)}"


def _sha256_bytes(payload: bytes) -> str:
    return hashlib.sha256(payload).hexdigest()


def build_execution_fingerprint(payload: dict[str, Any]) -> str:
    canonical = json.dumps(payload, sort_keys=True, separators=(",", ":")).encode()
    return _sha256_bytes(canonical)


def build_experiment_name(
    *,
    model_name: str,
    thinking_mode: str,
    prompt_family: str,
    prompt_version: str,
) -> str:
    ...


def build_experiment_id(
    *,
    experiment_name: str,
    comparison_coordinates: dict[str, Any],
) -> str:
    ...


def build_task_contract_sha(paths: Iterable[Path]) -> str:
    ...


def build_experiment_metadata(
    *,
    suite: str,
    dataset_name: str,
    dataset_path: Path,
    evaluators_path: Path,
    experiment_id: str,
    thinking_mode: str,
    model_name: str,
    prompt_family: str,
    prompt_version: str,
    prompt_sha: str,
    execution_fingerprint: str,
    batch_id: str,
) -> dict[str, Any]:
    return {
        "suite": suite,
        "dataset_name": dataset_name,
        "dataset_sha": _sha256_bytes(dataset_path.read_bytes()),
        "evaluator_contract_sha": _sha256_bytes(evaluators_path.read_bytes()),
        "experiment_id": experiment_id,
        "thinking_mode": thinking_mode,
        "model_name": model_name,
        "prompt_family": prompt_family,
        "prompt_version": prompt_version,
        "prompt_sha": prompt_sha,
        "execution_fingerprint": execution_fingerprint,
        "batch_id": batch_id,
    }
```

Implementation notes:

- keep `experiment_name` human-readable and stable across reruns of the same comparison coordinates
- derive the short hash in `experiment_id` from the comparison coordinates only, not from execution-only settings such as retries or concurrency
- compute `task_contract_sha` separately and include it in the execution fingerprint input even though it does not become a top-level metadata field
- do not add optional query-aid fields in this helper yet; the runners can merge them in explicitly
