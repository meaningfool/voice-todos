# Item 9.2 Transcript And Extraction Seam Refinement Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** `docs/superpowers/specs/2026-04-13-item9.2-transcript-extraction-seam-design.md`

**Goal:** Refine the backend seam above `SttSession` so transcript state is modeled as stable plus provisional text and extraction is triggered by endpoint signals or transcript change, not Soniox finalized-token mechanics.

**Architecture:** Keep the frontend websocket contract unchanged while reshaping the backend in three gated phases. Phase 1 introduces explicit transcript-state semantics, Phase 2 moves extraction to transcript-delta semantics, and Phase 3 removes remaining Soniox-shaped assumptions above the adapter and proves live provider viability through shared smoke tests. Acceptance gates are durable regression assets and are the only completion criteria for each phase.

**Tech Stack:** Python 3.11+, FastAPI websocket flow, pytest, existing STT abstraction in `backend/app/`, shared live smoke script `scripts/stt_ws_smoke.py`

## Phase 2: Transcript-Delta-Based Extraction Triggers

**Behavioral delta:** Non-endpoint extraction is driven by transcript growth
since the last extraction baseline, and final stop extraction is driven by
whether the final transcript changed since the last successful extraction.

### Task 2: Replace finalized-token-count extraction with transcript-delta semantics

**Files:**
- Modify: `backend/app/extraction_loop.py`
- Modify: `backend/app/transcript_accumulator.py`
- Modify: `backend/app/ws.py`
- Modify: `backend/tests/test_extraction_loop.py`
- Modify: `backend/tests/test_ws.py`

- [ ] **Step 1: Write the failing extraction verification tests**

Add failing tests in `backend/tests/test_extraction_loop.py` for:

- transcript-growth threshold calculation based on transcript snapshots
- no extraction before threshold
- extraction when threshold is crossed
- no redundant final extraction when stop transcript is unchanged
- exactly one final extraction when stop transcript changed

Add any narrow `/ws` tests needed in `backend/tests/test_ws.py` to prove the
backend flow still wires transcript-driven extraction correctly for Mistral.

Name the phase acceptance tests explicitly in `backend/tests/test_ws.py` as:

- `test_threshold_trigger_acceptance`

- [ ] **Step 3: Implement the minimal extraction-trigger refactor**
